# 🔬 Gemini API 测试指南

## 📋 测试前准备

### 1. 获取Google API Key
访问 [Google AI Studio](https://aistudio.google.com/app/apikey) 获取API Key

### 2. 安装/加载扩展
1. 打开Chrome浏览器，进入 `chrome://extensions/`
2. 开启"开发者模式"
3. 点击"加载已解压的扩展程序"
4. 选择 `ai-answer-assistant` 文件夹

### 3. 配置Gemini模型
1. 点击扩展图标打开侧边栏
2. 点击"⚙️ API设置"按钮
3. 点击"添加自定义模型"
4. 填写配置信息：

```
模型名称: Gemini 2.0 Flash (测试)
Base URL: https://api.google.com (任意URL)
API Key: 您的Google API Key
模型ID: gemini-2.0-flash
```

5. 点击"保存"
6. 在模型列表中选中该模型作为活跃模型

## 🧪 测试步骤

### 测试1: API连接测试

**步骤**:
1. 打开任意网页（确保不是系统页面）
2. 点击扩展图标 → "扫描页面题目"
3. 查看日志区域

**预期结果**:
- ✅ 日志显示"正在扫描题目..."
- ✅ 没有API错误信息
- ❌ 如果出现错误，查看[故障排除](#故障排除)部分

### 测试2: 实际答题测试

**步骤**:
1. 打开包含题目的网页（如问卷星、腾讯问卷等）
2. 点击"扫描页面题目"
3. 等待扫描完成（日志显示题目数量）
4. 点击"开始自动答题"

**预期结果**:
- ✅ 成功识别题目数量
- ✅ 自动填写答案
- ✅ 日志显示"已完成"状态

### 测试3: 不同Gemini模型测试

**测试配置**:
```
模型ID: gemini-pro
模型ID: gemini-1.5-pro
模型ID: gemini-1.5-flash
模型ID: gemini-2.0-flash
```

**预期结果**:
- ✅ 所有模型都能正常工作
- ✅ 响应速度和准确性差异

## 📊 验证方法

### 方法1: 浏览器控制台日志

**步骤**:
1. 按 `F12` 打开开发者工具
2. 切换到"Console"标签
3. 执行测试操作
4. 查看日志信息

**成功日志示例**:
```
[AI答题助手] Gemini API调用成功
[AI答题助手] 识别到 5 道题目
[AI答题助手] 答题完成
```

**错误日志示例**:
```
[AI答题助手] Gemini API请求失败 (401)
[AI答题助手] Gemini API Key无效或无权限
```

### 方法2: 扩展日志区域

**位置**: 扩展侧边栏底部"运行日志"区域

**成功日志**:
- "正在扫描题目..."
- "识别到 X 道题目"
- "开始自动答题"
- "已完成 X/X 题目"

### 方法3: 网络请求监控

**步骤**:
1. 按 `F12` → "Network"标签
2. 执行测试操作
3. 查找 `generativelanguage.googleapis.com` 请求

**验证要点**:
- ✅ 请求URL正确
- ✅ 响应状态为200
- ✅ 响应包含 `candidates` 字段

## 🔧 故障排除

### 问题1: "Gemini API Key无效"
**原因**: API Key错误或过期
**解决**:
1. 重新获取API Key
2. 检查API Key是否正确复制
3. 确认API Key格式（以`AIza`开头）

### 问题2: "Gemini: 请求格式错误"
**原因**: 模型ID不正确
**解决**:
- 使用正确的模型ID: `gemini-pro`, `gemini-1.5-pro`, `gemini-2.0-flash` 等
- 不要使用不存在的模型ID

### 问题3: "Gemini服务暂时不可用"
**原因**: 网络问题或服务故障
**解决**:
1. 检查网络连接
2. 等待几分钟后重试
3. 尝试使用代理服务

### 问题4: "响应被安全政策过滤"
**原因**: 题目内容触发安全过滤
**解决**:
1. 尝试其他题目
2. 使用不同的模型
3. 考虑使用代理服务

### 问题5: 无法识别题目
**原因**: 页面结构不支持或AI分析失败
**解决**:
1. 确认页面包含标准题目格式
2. 尝试刷新页面
3. 使用内置模板（问卷星、腾讯问卷）
